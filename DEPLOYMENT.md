# HBEONLABS Production Deployment Manual

This guide describes how to deploy the Employee Management System to your production server on **Port 3004** using **MySQL** and the **PM2 Process Manager**.

---

## Prerequisites

Before starting, ensure the following software is installed on your server:
1. **Node.js** (v18.x or v20.x recommended)
2. **pnpm** (preferred) or **npm**
3. **MySQL Server** (v8.x recommended)
4. **PM2** (Process Manager for Node.js)

To install PM2 globally:
```bash
npm install -g pm2
```

---

## Step 1: Create the MySQL Database

1. Connect to your MySQL Server instance via CLI:
   ```bash
   mysql -u root -p
   ```
   *(Enter your MySQL root password when prompted)*

2. Create the target production database with `utf8mb4` encoding to support rich text logging:
   ```sql
   CREATE DATABASE hbeonlabs_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. (Optional) Create a dedicated database user instead of using `root`:
   ```sql
   CREATE USER 'hbeon_user'@'localhost' IDENTIFIED BY 'your_secure_password';
   GRANT ALL PRIVILEGES ON hbeonlabs_db.* TO 'hbeon_user'@'localhost';
   FLUSH PRIVILEGES;
   ```
   Exit the MySQL prompt:
   ```sql
   EXIT;
   ```

---

## Step 2: Configure Environment Variables (`.env`)

In your server's deployment directory, create or edit the `.env` file. Change the parameters to match your database login:

```ini
# ==========================================
# DATABASE SETTINGS
# ==========================================
DB_DIALECT=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=hbeonlabs_db
DB_USER=root
# Change this to your actual MySQL user password
DB_PASSWORD=your_secure_password
# Keep SQLite file as a local debug fallback
DB_STORAGE=./database.sqlite

# ==========================================
# SERVER CONFIGURATION
# ==========================================
PORT=5000
NODE_ENV=production

# ==========================================
# SECURITY
# ==========================================
JWT_SECRET=use-a-long-random-alphanumeric-string-for-production
JWT_EXPIRY=7d

# ==========================================
# FRONTEND / NEXT.JS SETTINGS
# ==========================================
# Change localhost to your Server's Public IP or Domain name
NEXT_PUBLIC_API_URL=http://localhost:5000/api
```

---

## Step 3: Install Dependencies and Seed Database

1. Install project packages:
   ```bash
   pnpm install
   ```

2. Seed the database schemas and initialize default users (this will populate the database with designations, departments, and default test profiles with password **`Hbeonlabs@2026`**):
   ```bash
   pnpm seed
   ```

---

## Step 4: Build the Next.js Frontend

Compile the Next.js production bundles:
```bash
pnpm build
```

---

## Step 5: Launch Services using PM2

PM2 ensures the servers remain online, auto-restart upon crashes, and restart automatically during server reboots.

1. **Start the Backend Server (Express API)**:
   ```bash
   pm2 start server/server.js --name "hbeonlabs-backend"
   ```

2. **Start the Frontend Server (Next.js) on Port 3004**:
   In `package.json`, the `"start"` script is preconfigured to launch Next.js on port 3004:
   ```bash
   # package.json script check
   # "start": "next start -p 3004"
   
   pm2 start pnpm --name "hbeonlabs-frontend" -- start
   ```

3. **Save PM2 processes state**:
   ```bash
   pm2 save
   ```

4. **Enable system startup hook**:
   Generates scripts to start PM2 services automatically after system reboots:
   ```bash
   pm2 startup
   ```
   *(Copy and paste the command output generated by PM2 to finalize setup)*

---

## Step 6: Verify Deployment

* **Frontend Panel**: Open `http://<your-server-ip>:3004`
* **API Endpoints**: Open `http://<your-server-ip>:5000/health`

## Useful PM2 Commands

* Check logs in real-time:
  ```bash
  pm2 logs
  ```
* Check services health metrics:
  ```bash
  pm2 status
  ```
* Restart services:
  ```bash
  pm2 restart all
  ```
